Método incremental basado en un plan

Este desarrollo parte de un desarrollo parcial(un bosquejo) y de ahí ir agregando las funciones, por supuesto recibiendo siempre la retroalimentación del cliente. En general, las primeras funciones en agregarse son las primordiales del sistema. Se dice que es basado en un plan porque los incrementos se establecen desde el principio, a diferencia del ágil, que se tocará más adelante.

Además en este método hay una relación directa entre las versiones y el desarrollo, especificación y validación. Estas etapas corresponden con las definidas anteriormente para el método en cascada, pero aquí su relación es más directa e inmediata. El desarrollo de software incremental es más acorde a cómo se resuelven los problemas, pues comúnmente no se tiene una solución completa a un problema.

Entre sus ventajas se reduce la dificultad de modificar los requerimientos ya avanzado el proyectos, además el cliente se beneficia de una alta retroalimentación de cada una de las etapas del proceso inclusive los clientes pueden empezar a utilizar el producto aún sin terminar para así asegurarse de que es lo que requiere.

Sin embargo, este sistema tiene la desventaja de que para el coordinador del proyecto es difícil medir los tiempos. Además cada que se agregan funciones el sistema se vuelve más y más pesado pues se desarrolla encima de otra capa en lugar de linealmente. En los sistemas críticos o en proyectos muy grandes este método sufre de estos dos problemas de manera muy clara y no se recomienda para este tipo de proyectos.
