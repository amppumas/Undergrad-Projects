Protección

La protección se encarga de evitar que el sistema falle por algún tipo de actividad intencional, no tiene relación con los accidentes como es el caso de la seguridad. Es un término que muchas veces se confunde con la seguridad y es considerado por algunos profesionales como lo mismo que la seguridad, en este tutorial nos referiremos a la protección como la propiedad para evitar ataques intencionales.

Hay 4 enfoques que se pueden usar para conocer el nivel de protección del sistema:

  1. Se analiza el sistema y usando la experiencia de ataques pasados se hacen pruebas en el sistema.
  2. Se usan herramientas que tiene como objetivo romper con la protección del sistema.
  3. Se usan pruebas formales orientadas a verificar la protección.
  4. Los equipos tigre, que son conformados por grupos de personas que tienen como objetivo romper la protección del sistema.

La protección es muy difícil de verificar, sobretodo con la gran cantidad de sistemas que deben tener acceso a internet. Dependiendo de la región en donde se desarrolle el sistema hay diferentes criterios que se usan para conocer el nivel de seguridad de un sistema, por lo que si se tiene que tener un nivel en específico de protección se puede dejar en manos de evaluadores especializados en el tema para conocer que características necesita el sistema para cumplir con el nivel requerido.

Se debe recordar que no se debe usar ningún componente que no haya sido verificado, ya que un componente sin verificar puede ser la entrada a cualquier tipo de ataque.
