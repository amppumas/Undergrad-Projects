Método basado en la reutilización

De una manera u otra es común que nos encontremos con la reutilización de software en los sistemas computacionales actuales ya sean procesadores de texto, servicios web estándar, o módulos que previamente se hayan desarrollado. Inclusive las empresas actuales tienen un repositorio de componentes de software para su reutilización. Existe formalmente un método de desarrollo de software que se basa exclusivamente en la utilización. Aunque en la práctica este sea mezclado con otras estrategias. En la figura se muestran las etapas a seguir para este método, tanto la primera como la última ya fueron definidas anteriormente, pero las demás se definen de la siguiente manera:

  1. Análisis de componentes: En cuanto se termina la fase de especificación se buscan los componentes que más se parezcan a lo que se necesita para el sistema

  2. Modificación de los requerimientos: Una vez que ya se tienen los componentes se actualizan los requerimientos para mostrar que se logró y si se necesita se regresa a la etapa anterior para evaluar las posibilidades.

  3. Diseño de sistema con reutilización: En esta etapa se diseña el sistema utilizando los componentes adquiridos y se diseñan los que se requieran para cumplir con los requerimientos.

  4. Desarrollo e Integración: Ya que se tienen todos los módulos diseñados se implementan y se unen para formar el sistema completo, ya utilizando los componentes reutilizados y los diseñados

Las ventajas de este método son que reduce considerablemente el costo del sistema así como los riesgos y todo esto deriva en una entrega mucho más rápida. Sin embargo se pierde un poco el control del sistema y además puede ser que no se cumplan todos los requerimientos del cliente.
